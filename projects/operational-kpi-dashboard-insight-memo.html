KPI Definition Table

KPI Formula Decision Use
Average Resolution Time AVG(resolved_at - created_at) Baseline speed and staffing calibration
Median Resolution Time PERCENTILE_CONT(0.5) within resolved durations Outlier-resistant service speed signal
SLA Breach % 100 * SUM(sla_breach_flag) / COUNT(*) Reliability and risk monitoring
Backlog Size COUNT(*) WHERE status <> 'Closed' Queue congestion and execution pressure
Reopen % 100 * SUM(reopened_flag) / COUNT(*) First-time fix quality indicator
Repeat Ticket % 100 * SUM(repeat_ticket_flag) / COUNT(*) Root-cause and knowledge-gap detection

How These Numbers Are Computed

KPI values are generated from support_tickets using SQL in operational-kpi-analysis-pack.sql. Use these snippets to calculate dashboard-ready outputs. 

-- 1) Average resolution time (hours)
SELECT ROUND(AVG(EXTRACT(EPOCH FROM (resolved_at - created_at)) / 3600.0), 2) AS avg_resolution_hours
FROM support_tickets
WHERE resolved_at IS NOT NULL;

-- 2) Median resolution time (hours)
SELECT ROUND(
  PERCENTILE_CONT(0.5) WITHIN GROUP (
    ORDER BY EXTRACT(EPOCH FROM (resolved_at - created_at)) / 3600.0
  ),
  2
) AS median_resolution_hours
FROM support_tickets
WHERE resolved_at IS NOT NULL;

-- 3) SLA breach percentage
SELECT ROUND(100.0 * AVG(sla_breach_flag), 2) AS sla_breach_pct
FROM support_tickets;

-- 4) Backlog size (open and in progress)
SELECT COUNT(*) AS backlog_size
FROM support_tickets
WHERE status <> 'Closed';

-- 5) Reopen and repeat ticket percentages
SELECT
  ROUND(100.0 * AVG(reopened_flag), 2) AS reopen_rate_pct,
  ROUND(100.0 * AVG(repeat_ticket_flag), 2) AS repeat_ticket_rate_pct
FROM support_tickets;

Executive Insight Memo

Date: February 18, 2026

To: Operations Leadership

From: Business Systems Analyst

Subject: Service Activation and Support Efficiency KPI Review

KPI evidence shows service risk is concentrated in higher-priority work. Breach rates are materially higher in High and Urgent queues, and backlog aging indicates unresolved items exceeding 72 hours. Resolution speed is uneven by category, with Provisioning and Technical queues driving most delay pressure.

Quality indicators reinforce the same risk profile: categories with longer resolution duration show elevated reopen and repeat rates, and CSAT is lower when SLA breaches occur. This pattern suggests that throughput and first-time fix quality need to be improved together rather than as separate initiatives.

Recommended actions:

  1. Set daily ownership for aged backlog triage and enforce 72-hour exception review.
  2. Apply targeted coaching and playbooks in high-breach category-priority combinations.
  3. Rebalance staffing and escalation coverage for peak weekly load windows.
  4. Track weekly KPI deltas in leadership reviews using the defined dashboard layout.
