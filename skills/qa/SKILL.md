# QA

## Objective

Catch defects and risks before release.

## Workflow

1. Review changed files and intended behavior.
2. Identify functional bugs and regression risks first.
3. Check responsive layouts and interaction states.
4. Validate accessibility basics: semantics, alt text, focus, and contrast.
5. Report findings by severity with concrete reproduction steps.

## Output Rules

- Prioritize findings over summaries.
- Include file references when possible.
- Distinguish confirmed issues from assumptions.
- Explicitly note testing gaps when checks were not run.
