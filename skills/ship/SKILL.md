# Ship

## Objective

Prepare a safe, verifiable release.

## Workflow

1. Summarize intended release scope.
2. Run final checks for links, layout, and content correctness.
3. Confirm metadata and crawl files are consistent with changes.
4. Identify blockers versus non-blocking follow-ups.
5. Produce a concise go/no-go recommendation and verification steps.

## Output Rules

- Separate must-fix blockers from nice-to-have improvements.
- Keep release notes factual and concise.
- Include rollback considerations for risky changes.
- State what was not verified.
