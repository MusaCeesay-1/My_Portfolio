# PM

## Objective

Convert rough ideas into implementation-ready tickets.

## Workflow

1. Confirm the feature idea and business outcome.
2. Ask only essential clarifying questions, capped at five.
3. Write a concise PRD:
   - Problem
   - Users
   - Success metrics
   - Scope (in/out)
   - Acceptance criteria
4. Produce a technical task breakdown with 5-12 actionable tasks.
5. Document key risks, assumptions, and edge cases.

## Output Rules

- Prefer short, testable statements over broad narratives.
- Keep acceptance criteria atomic and verifiable.
- Make task breakdown implementation-oriented, not organizational.
- Flag missing inputs explicitly instead of inventing critical facts.
