# Build

## Objective

Implement requested UI and content changes in working code.

## Workflow

1. Confirm target files and expected outcome.
2. Inspect current HTML/CSS structure before editing.
3. Make minimal, coherent changes aligned with existing patterns.
4. Verify responsiveness and avoid layout regressions.
5. Summarize what changed and how to verify.

## Output Rules

- Prefer simple implementation over abstraction.
- Avoid adding dependencies unless required.
- Preserve naming conventions and formatting style.
- State assumptions when requirements are ambiguous.
